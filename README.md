# ジオゲッサー風オリジナルゲーム 開発仕様書

## 1. プロジェクト概要
* **目的:** Googleストリートビューを活用し、表示された場所を地図上で当てるブラウザゲームの作成。
* **ターゲット:** 中学生（授業の導入やレクリエーションで直感的に遊べる難易度・操作性）。
* **公開環境:** GitHub Pages（HTML, CSS, JavaScriptによる静的サイト）。
* **連携:** 完成後は既存のゲームポータルサイトにリンクを追加する。タイトル画面の開始ボタン表記は、他のゲームと統一して「Play」とする。

## 2. 必須準備（Google Cloud 設定）
* [ ] プロジェクトの作成と請求先アカウントの登録。
* [ ] `Maps JavaScript API` の有効化。
* [ ] APIキーの発行。
* [ ] **【重要：課金防止】** 「1日あたりのAPIリクエスト回数上限（クォータ）」を安全な回数に設定する。
* [ ] **【重要：不正利用防止】** APIキーに「HTTPリファラー制限」をかけ、自身のGitHub PagesのURLからしかAPIを叩けないようにする。

## 3. ゲームの基本システム
1. **出題:** コード内に配列で用意した座標（緯度・経度）リストからランダムに1箇所を選び、ストリートビューを表示する。
2. **解答:** 画面上に配置した解答用マップをクリックしてピンを刺し、「決定」ボタンを押す。
3. **判定:** 正解の出題座標と、プレイヤーがピンを刺した座標の「2点間の直線距離」を算出する。
4. **スコア:** 距離が近いほど高得点になる計算式を適用し、結果画面に表示する。

## 4. 開発ステップ（AIへの指示手順）
* **Step 1:** APIキーを読み込み、画面の左側にストリートビュー、右側に解答用マップを同時に表示するベースのHTML/CSS/JavaScriptを作成する。
* **Step 2:** 右側の解答用マップをクリックした際、その場所にマーカー（ピン）を設置・移動できる機能を実装する。
* **Step 3:** ダミーの出題座標（例：わかりやすいランドマーク）を設定し、ストリートビューをその場所に連動させる。
* **Step 4:** 解答ボタンを押した際に、正解座標とマーカー座標の距離（km）を計算し、画面に表示するロジックを組む。
* **Step 5:** UIデザインを整え、タイトル画面からプレイ画面、結果発表画面への遷移を作成する。
